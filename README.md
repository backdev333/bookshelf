# Bookshelf — Microservices

## Почему микросервисы?

Монолит стал узким местом: 

То что начиналось как небольшой пет-проект начало превращаться в популярное приложение. Команда аналитиков выявила разную востребованность возможностей нашего монолита, как оказалось что пользователи очень много используют функционал взаимодействия с каталогом книг и связанными сущностями, а функционал регистрации/авторизации напротив, используется значительно реже.

Вместе с ростом приложения росла и команда, которая в конце концов разделилась на две. Работать двум независимым командам в общем репозитории оказалось не просто.

Так как одну из частей нашего приложения клиенты использовали чаще, команде чаще приходилось внедрять туда новый функционал соответствующий запросам потребителя, в связи с чем появилась необходимость в раздельном деплое приложения.


## Архитектура

- **auth-service** (порт 8081) — регистрация, авторизация, управление пользователями
- **books-service** (порт 8082) — каталог книг, рецензии

Каждый сервис имеет свою базу данных (Database per Service).

## Компоненты системы

| Компонент | Назначение |
|-----------|-----------|
| auth-service | Аутентификация и пользователи |
| books-service | Книги и рецензии |
| auth-postgres | БД для auth-service |
| books-postgres | БД для books-service |
| frontend | React-приложение |
